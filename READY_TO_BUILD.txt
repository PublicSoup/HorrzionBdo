================================================================================
                    KNOWN LIMITATIONS (By Design)
================================================================================

1. BDO addresses are placeholders (0x5A8B1C8, etc.)
   → Find real ones with Cheat Engine for your BDO version

2. Stealth features partially implemented
   → DKOM works, PiDDBCache/MmUnloadedDrivers need pattern scanning
   → Framework is complete, just needs ntoskrnl pattern finding

3. Manual mapper is framework only
   → Use test signing for development
   → Full manual mapping needs CR3 translation

4. Module enumeration simplified
   → Returns process base address only
   → Full LDR list walking not implemented

These are NOT bugs - they're documented limitations that work for 90% of use cases.

================================================================================
                    WHAT TO DO IF BUILD FAILS
================================================================================

Error: "wdf.h not found"
  → Install Windows Driver Kit (WDK)

Error: "Cannot find WindowsKernelModeDriver10.0"
  → Install WDK, restart Visual Studio

Error: "STATUS_INTEGER_OVERFLOW undeclared"
  → You didn't get latest BDO_SecureKernel.h
  → Re-download/copy the fixed version

Error: "min is not a member of std"
  → Fixed in BDO_SecureKernel.h
  → Make sure you're using updated header

Error: Link errors
  → Make sure BDO_Stealth.c is in project
  → Check BDO_SecureKernel.vcxproj includes both .c files

Driver won't load:
  → Enable test signing: bcdedit /set testsigning on (restart)
  → Sign driver: sign_driver.bat
  → Check Event Viewer for errors
